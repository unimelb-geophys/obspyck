FOCMEC applied to a data set of P-wave polarities

This page supplements the discussion in section IV A of the FOCMEC manual: the application of the FOCMEC package to find the focal mechanism for a magnitude 4.2 event using P waves recorded by 37 local-to-regional-distance stations. Directory ./sample_runs/ETNEvent includes a README file, drivers, and input files for the analysis. Copies of output files are in directory ./doc/ETNEvent-doc/M/i>

The driver files for programs Focmec and Focplt are Unix csh scripts that include comments. The programs can be run interactively without using scripts, but using the scripts allows quick editing to fine-tune runs. For Focmec, the first run was done with eight unity-weighted polarity errors. An examination of the output file indicated that four was the minimum that could give a well-constrained solution.

As with the sample runs for the Sakhalin-Island event, the Focplt driver scripts as written end with call to produce first a screen display of the plot and then to convert the SGF plot file to a PDF file. See Appendix D in the manual for more details on graphics in the FOCMEC package.

Prior to March, 2002, program Focmec input code allowed one to enter + and - for emergent polarities (compression and dilatation respectively), but it did not use them as data. (However, they could be plotted in program Focplt, if the user so chose.) A user suggested there be an option for them to be used in program Focmec. Accordingly, if any “SENSE” is + or -, the user will be prompted to choose if emergent polarity picks should be used or ignored. To demonstrate this, a set of files has been added: an input data file in which three of the P picks have been changed from “C” to “+” plus two driver files for the two options. The new files all have “emergence” in their filespecs.
